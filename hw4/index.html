Overview

In this homework I implemented a real-time cloth simulation using a mass-spring system. I built the cloth mesh with point masses and springs (structural, shearing, bending), simulated physics via Verlet integration with external and spring forces, handled collisions with spheres and planes, implemented self-collision using spatial hashing, and wrote several GLSL shaders including diffuse, Blinn-Phong, texture mapping, bump mapping, displacement mapping, mirror reflection, and a custom toon/cel shader. Perhaps the most fascinating thing I learned during this project was how heavily the illusion of realistic physical materials relies on microscopic parameter tuning—seeing firsthand how slight adjustments to spring stiffness, density, or numerical damping completely transform the macroscopic draping, wrinkling, and folding behavior of the cloth.

Part 2: Simulation via Numerical Integration

Effect of Spring Constant (ks)

With higher ks values the cloth doesn't sag downwards too much and is much more stiff/resistant to the force. We don't see the bend of the pinning so much as shown in this example with a ks value of 15000. With ks value of 100, the cloth sags significantly more and is much more flexible.

Effect of Density

Density follows the opposite trend of ks. Higher values of density make the cloth sag much more while lower values make it more stiff/less bendable.

Effect of Damping

One way the cloth loses energy is by damping. Higher damping values make the cloth stop moving and oscillating faster, which makes it settle faster but could also stop some of the more fluid and realistic behaviors. Lower damping lets the cloth move around more freely, which can make the movements more realistic and dynamic, but it may also make the cloth take longer to stop moving. The value of the damping should depend on the kind of cloth you want to look like.

Part 4: Handling Self-Collisions

Implementation

To implement self-collision we go through all the point masses of the colliding object and collide the objects of our current point mass with the colliding object except our own point mass. To store our entire grid and to make sure the run time of this collision isn't O(n²) but O(n) we use a hashing system and hence divide the grid into smaller sections and only check each point with all the other points in its grid/section.

Part 5: Shaders

Shader Program Explanation

A shader program is a set of small programs that run on the GPU in parallel, processing vertices and fragments (pixels) to determine how objects appear on screen. Vertex shaders and fragment shaders work together in a pipeline: the vertex shader runs first, transforming each vertex's 3D position into screen-space coordinates and passing along interpolated data (like normals, UV coordinates, and world-space positions). The fragment shader then runs for every pixel covered by a triangle, using the interpolated values from the vertex shader to compute the final color of that pixel. Together, they create lighting and material effects — the vertex shader sets up the geometry and per-vertex data, while the fragment shader computes per-pixel lighting, texturing, and shading.

Blinn-Phong Shading Model

The Blinn-Phong shading model computes lighting as the sum of three components:

  • Ambient: A constant base illumination so nothing is fully black
  • Diffuse: Light scattered equally in all directions based on the angle between the surface normal and light direction
  • Specular: A bright highlight based on the half-vector between the view and light directions, raised to a power to control shininess

The formula is: \( L = k_a I_a + k_d (I/r^2) \max(0, \mathbf{n} \cdot \mathbf{l}) + k_s (I/r^2) \max(0, \mathbf{n} \cdot \mathbf{h})^p \)

Bump mapping only perturbs the surface normals to create the illusion of detail without changing the geometry, while displacement mapping actually moves the vertices based on the height map, creating real geometric detail. With bump mapping, the silhouette of the sphere remains smooth; with displacement mapping, you can see the sphere's outline actually deformed.

Custom Shader

My custom shader makes a cel-shading or toon effect. It splits the Blinn-Phong diffuse lighting into four bands to create the flat, cartoon-like shading that is common in cel animation. Instead of a smooth falloff, the specular highlight uses a hard binary threshold for a more stylized look. I added rim lighting based on Fresnel that makes a blue glow around the edges of silhouettes. This is where the surface normal is almost perpendicular to the view direction. The colors change from warm orange in the light to cool blue in the dark. Finally, edges where dot(n, v) < 0.15 are made to look almost black. This makes the object's outline strokes stand out.

Extra Credit

1. Spatially-Varying Wind Force

For extra credit I implemented a spatially-varying wind force that creates dynamic, rippling cloth motion. Instead of a uniform external force, the wind varies across the cloth surface and over time using sinusoidal functions:

// Wind direction and strength
Vector3D wind_dir(0.5, 0.0, 1.0);
double wind_strength = 15.0;
// Per-point-mass spatially-varying gust
double spatial_noise = sin(2.0 * pm.position.x + wind_time * 3.0) *
                       cos(1.5 * pm.position.y + wind_time * 2.0);
double gust = 0.5 + 0.5 * spatial_noise;
pm.forces += wind_dir * wind_strength * gust * mass;

The wind force is applied in Cloth::simulate() after the external accelerations (gravity) and before the spring forces. The key features are:

  • Spatial variation: The wind strength varies based on each point mass's position, creating localized gusts rather than uniform push. This uses sin(2x + 3t) × cos(1.5y + 2t) to produce wave-like patterns across the cloth.
  • Temporal variation: The phase offsets (wind_time * 3.0 and wind_time * 2.0) cause the gust pattern to travel across the cloth over time, producing a natural rippling/flapping effect.
  • Directional: The wind blows primarily in the +z direction with a slight +x component, pushing the cloth outward and to the side.

The result is that the cloth ripples and billows realistically as if being blown by a gentle breeze, with different parts of the cloth responding differently to the wind at any given moment. This is a significant visual improvement over the static gravity-only simulation.

2. Box Collision Primitive

I added a new 3D collision primitive — an axis-aligned box (rectangular prism). This required creating new source files (collision/box.h and collision/box.cpp), integrating them into the build system, and adding JSON scene parsing so box objects can be loaded from scene files.

The collision algorithm works as follows:

  1. Check if the point mass is inside the box's axis-aligned bounding volume.
  2. If inside, compute the penetration depth to each of the 6 faces (±X, ±Y, ±Z).
  3. Find the face with minimum penetration distance — this is the closest exit face.
  4. Push the point mass out through that face with a surface offset, then apply friction correction (same pattern as the sphere and plane collisions).
// Find minimum penetration depth among 6 faces
double dx_min = pos.x - min_corner.x;
double dx_max = max_corner.x - pos.x;
// ... (same for dy, dz)
// Push out through nearest face
Vector3D tangent_point = pos + normal * (min_dist + SURFACE_OFFSET);
Vector3D correction = tangent_point - pm.last_position;
pm.position = pm.last_position + (1.0 - friction) * correction;

The box also renders as 12 triangles (6 faces × 2 triangles each) with proper normals for shading. A new scene file scene/box.json places the box at the center of the simulation space so the cloth falls and drapes over it.
